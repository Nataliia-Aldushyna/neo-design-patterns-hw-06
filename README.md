# 💬 Message Service (TypeScript)

## Домашнє завдання до теми: Структурні патерни — Decorator та Proxy

---

## 📌 Опис завдання

У цьому домашньому завданні було реалізовано спрощений сервіс повідомлень із використанням структурних патернів **Decorator** та **Proxy**.

Застосунок демонструє, як можна розширювати функціональність сервісу без зміни його коду та контролювати доступ до нього через додатковий рівень абстракції.

Основною метою роботи було:

- навчитися розширювати поведінку об'єктів через декоратори;
- реалізувати контроль доступу через проксі;
- забезпечити роботу через абстракцію `IMessageService`;
- продемонструвати принцип Open/Closed (OCP).

---

## 🎯 Мета роботи

Реалізувати консольну утиліту, яка:

- надсилає повідомлення через базовий сервіс;
- автоматично додає часову мітку до повідомлення;
- перетворює текст повідомлення у верхній регістр;
- обмежує частоту надсилання повідомлень;
- блокує спроби спаму;
- працює через єдиний інтерфейс `IMessageService`.

---

## 🏗 Реалізована функціональність

Система підтримує:

- надсилання повідомлень через `MessageService`;
- додавання часової мітки через декоратор `@withTimestamp`;
- перетворення тексту у верхній регістр через декоратор `@uppercase`;
- обмеження частоти викликів через `RateLimitProxy`;
- блокування повідомлень, які надсилаються раніше дозволеного інтервалу;
- легке розширення новими декораторами або проксі без зміни існуючого коду.

---

## 🧠 Застосовані патерни

### ✔ Decorator (Декоратор)

Патерн Decorator використовується для розширення функціональності методу `send()` без зміни його реалізації.

Реалізовані декоратори:

- `@withTimestamp`
- `@uppercase`

Ланцюг обробки повідомлення:

```txt
Повідомлення
      ↓
@withTimestamp
      ↓
@uppercase
      ↓
MessageService.send()
```

Результат:

```txt
[2026-05-29 20:00:02] ПРИВІТ! ЯК СПРАВИ?
```

---

### ✔ Proxy (Замісник)

Патерн Proxy використовується для контролю доступу до сервісу повідомлень.

Реалізований компонент:

```txt
createRateLimitProxy()
```

Proxy:

- відстежує час останнього успішного виклику;
- блокує надсилання повідомлень частіше встановленого інтервалу;
- не змінює логіку самого MessageService.

Схема роботи:

```txt
Client
   ↓
RateLimitProxy
   ↓
MessageService
```

## ℹ️ Примітка

Проєкт було реалізовано з використанням TypeScript 6.0.3.

Нова версія TypeScript підтримує сучасний синтаксис декораторів, тому параметр:

```json
"experimentalDecorators": true
```

не використовується, оскільки він призводить до конфлікту зі стандартною реалізацією декораторів у TypeScript 6.x.
